Important FAQ

1. What is the Document Object Model (DOM)?

The **Document Object Model (DOM)** is a programming interface for HTML and XML documents. It represents the document as a tree of nodes, where each element is an object. JavaScript can use the DOM to manipulate and interact with these nodes, allowing dynamic changes to content, structure, and styles. The DOM is a live representation, meaning changes to the document are immediately reflected. Itâ€™s essential for handling events like clicks and form submissions in web development.

2. How do you select an element from the DOM?

We can select an item in multiple ways.such as:
1. document.getElementById(): Selects an element by its id.
2. document.querySelector(): Selects the first element that matches a CSS selector.
3. document.querySelectorAll(): Selects all elements matching a CSS selector.
4. document.getElementsByClassName(): Selects elements by their class name.
5. document.getElementsByTagName(): Selects elements by their tag name.

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where you attach a single event listener to a parent element rather than each child element. This works because events in the DOM propagate (or "bubble") up through the ancestor elements.
Why it's useful:
1. Performance:: Instead of adding event listeners to multiple child elements, one listener on the parent can handle them all, reducing memory usage.
2. Dynamic content: It allows event handling for dynamically added elements, as the parent can capture events even for children added after the event listener was attached.
3. Simpler code: Fewer event listeners mean cleaner, more maintainable code.

4. How do you manipulate an element's attributes and styles using the DOM?

You can manipulate an element's attributes and styles using the DOM as follows:
1. Attributes:
- setAttribute(): Sets or updates an attribute.
- getAttribute(): Retrieves an attribute's value.
- removeAttribute(): Removes an attribute.
2. Styles:
- style: Modifies inline CSS styles.
- classList.add() / classList.remove()`: Adds or removes CSS classes.
